# ATLAS AI — Industrial Knowledge Intelligence Platform

> *"The AI Brain for Industrial Operations"*

A working hackathon prototype that ingests industrial documents, builds a
knowledge graph, and reasons over assets, maintenance history, incidents,
and regulations — instead of just answering questions like a chatbot.

Not a RAG chatbot. Not a dashboard. A connected reasoning engine:
**Documents → Knowledge Graph → GraphRAG Copilot → Predictive Maintenance
→ Root Cause Analysis**, all in one Next.js app.

## Runs with zero external accounts

This app works immediately after `npm install` — no Supabase, Neo4j,
Qdrant, or OpenRouter account required:

- **Data**: a file-backed JSON repository (`/data`) stands in for
  Postgres. Swap it for real Supabase in ~30 minutes — see
  `lib/db/README.md`.
- **Embeddings**: an offline hashing-trick vectorizer gives meaningful
  similarity search over industrial text with no API calls. Point
  `EMBEDDINGS_API_URL` at BGE-M3/OpenAI/Voyage to upgrade it.
- **LLM reasoning**: without `OPENROUTER_API_KEY`, a deterministic
  "reasoning composer" builds grounded answers directly from retrieved
  evidence, so every feature still demos convincingly. Add the key and
  every answer is generated by a real model (Claude, Gemini, DeepSeek,
  Llama, or Qwen — configurable via `OPENROUTER_MODEL`).

This means judges can clone, `npm install`, `npm run dev`, and see the
whole platform work — then you flip on real AI mid-demo by pasting in an
OpenRouter key.

## Quick start

```bash
npm install
cp .env.example .env.local   # optional — add OPENROUTER_API_KEY to go live
npm run dev
```

Open http://localhost:3000. The landing page links to the platform;
the demo dataset (6 assets, 7 documents, incidents, maintenance history,
compliance items) seeds itself automatically on first API call. You can
reset it anytime with the **"Reset demo data"** button in the top bar,
or `POST /api/seed`.

## Architecture

```
Documents (PDF/TXT/paste)
   |
   v
Document Intelligence Agent  (lib/agents/documentAgent.ts)
   |  OCR hook -> chunk -> embed
   v
Entity Extraction Agent      (lib/agents/entityAgent.ts)
   |  asset codes, technicians, regulations, parts
   v
Knowledge Graph Agent        (lib/agents/graphAgent.ts)
   |  nodes + relationships
   v
GraphRAG Retriever           (lib/graphrag.ts)
   |  vector similarity + 1-hop graph traversal
   v
+-------------+----------------+---------------+-------------------+
| RAG/Copilot | Maintenance    | Root Cause    | Executive         |
| Agent       | Agent          | Agent         | Summary Agent     |
| (citations, | (risk scoring, | (timeline,    | (dashboard        |
| confidence) | RUL forecast)  | evidence,RCA) | insights)         |
+-------------+----------------+---------------+-------------------+
   |
   v
OpenRouter (Claude / Gemini / DeepSeek / Llama / Qwen)
   or offline reasoning composer if no API key
```

All agents live in `lib/agents/` and are orchestrated by
`lib/agents/coordinator.ts` (the ingestion pipeline) — a lightweight
TypeScript DAG runner in place of a Python/LangGraph worker, since this
build targets a single deployable Next.js app. See "Upgrading to the full
stack" below for the Python/Celery upgrade path.

### Repository layer (swap-ready for Supabase)

Every read/write in the app goes through `lib/store.ts`, a typed
repository over `lib/db/jsonStore.ts` (local JSON files). This mirrors
the table shapes in `supabase/schema.sql` 1:1, so moving to real
Postgres/Supabase is a matter of re-implementing the functions in
`lib/store.ts` — nothing in the agents, API routes, or UI needs to
change. Full instructions in `lib/db/README.md`.

## Feature tour

| Module | Route | What it does |
|---|---|---|
| Landing | `/` | Product pitch with a live "reasoning trace" of the Compressor C-102 scenario |
| Executive Dashboard | `/dashboard` | KPIs, AI-generated insights, critical asset table |
| Document Intelligence | `/documents` | Drag-and-drop or paste-to-ingest, live pipeline progress, entity/chunk viewer |
| Knowledge Graph | `/graph` | Interactive React Flow graph, click-to-inspect connections |
| AI Copilot | `/copilot` | GraphRAG chat with citations, confidence score, reasoning trace, suggested questions |
| Asset Explorer | `/assets`, `/assets/[id]` | Digital profile, RUL forecast chart, maintenance timeline, AI recommendation |
| Root Cause Analysis | `/rca` | Pick an incident, get a generated timeline, evidence, contributing factors, corrective/preventive actions, printable report |
| Compliance Center | `/compliance` | Regulation tracker (Factory Act / ISO / OSHA / OISD), audit score |
| Digital Twin | `/twin` | Color-coded 2D plant layout, click an asset for a live snapshot |

## Demo script (~4 minutes)

1. **Documents** — paste the sample inspection report text (or drop a
   PDF) and watch the pipeline stages complete live.
2. **Knowledge Graph** — open the graph, click the `Compressor C-102`
   node, show its connections to technicians, the 2022 incident, and
   OISD-STD-137.
3. **Copilot** — ask *"Why is Compressor C-102 at risk?"* — point out the
   citations, confidence score, and expandable reasoning trace.
4. **Asset Explorer** — open C-102's digital profile, show the RUL
   forecast chart and AI recommendation.
5. **Root Cause Analysis** — generate the RCA report for the 2026
   overheating incident; show the timeline and corrective/preventive
   actions; hit Export/Print.
6. **Digital Twin** — close on the plant layout with C-102 pulsing red.

## Tech stack

Next.js 15 (App Router), React 19, TypeScript, Tailwind CSS v4,
Framer Motion, React Flow, Recharts, Lucide Icons, OpenRouter,
pdf-parse. See `.env.example` for every optional integration hook
(embeddings, OCR, Supabase, Neo4j, Qdrant).

## Upgrading to the full stack described in the original brief

This build intentionally trades the full Python/FastAPI/Celery/Neo4j/
Qdrant stack for a single deployable Next.js app, so it runs anywhere
with zero infra. To grow it into the full architecture:

1. **Supabase** — run `supabase/schema.sql`, swap `lib/store.ts` (see
   `lib/db/README.md`).
2. **Qdrant** — replace `lib/embeddings.ts`'s local vectorizer with a
   Qdrant client once `QDRANT_URL`/`QDRANT_API_KEY` are set.
3. **Neo4j Aura** — mirror writes from `lib/agents/graphAgent.ts` into
   Cypher `MERGE` statements for real graph traversal at scale.
4. **FastAPI + Celery workers** — move `lib/agents/coordinator.ts`'s
   ingestion pipeline into a Python worker queue if upload volume/latency
   requires it; the API contract (`POST /api/documents`) stays the same.
5. **Docling/PaddleOCR** — point `OCR_API_URL` at a hosted OCR service
   for scanned image ingestion.

## Deployment

- **Vercel**: `vercel deploy` — works out of the box in demo mode. Add
  `OPENROUTER_API_KEY` (and any of the optional vars above) in the
  Vercel project's environment variables.
- Note: the local JSON store persists to the filesystem, which is
  ephemeral on serverless platforms. For a persistent multi-user demo,
  connect Supabase (see above) before deploying publicly.
